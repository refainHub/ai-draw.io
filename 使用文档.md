## 快速开始

### 第一步：安装与启动

**前置要求：**
- 已安装 Node.js（版本 18 或更高）
- 已安装 npm

**安装步骤：**

```bash
# 进入项目目录
cd your-project-folder

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

**启动成功后，打开浏览器访问：**

```
http://127.0.0.1:6002
```

> 【截图位置：启动成功后的浏览器界面】

---

### 第二步：配置 AI 模型

首次使用需要配置 AI 模型的 API 密钥。

**配置步骤：**

1. 点击右侧聊天面板右上角的 **设置图标** ⚙️
2. 在设置对话框中点击 **"API 密钥和模型"** 选项
3. 填写以下信息：
   - **API 密钥**：您的 AI 服务 API Key
   - **基础 URL**（可选）：自定义 API 端点地址
   - **模型 ID**：要使用的模型名称（如 `gpt-4o`、`claude-sonnet-4-6` 等）

4. 点击 **"测试"** 按钮验证配置是否正确
5. 测试成功后点击 **"保存"**

> 【截图位置：模型配置对话框界面】

**支持的 AI 服务商：**
- OpenAI
- Anthropic（Claude）
- Google（Gemini）
- DeepSeek
- SiliconFlow
- ModelScope
- MiniMax
- GLM
- Qwen
- Kimi
- 以及其他兼容 OpenAI API 的服务

> ⚠️ **重要提示：** API 密钥仅存储在您的浏览器本地，不会上传到服务器。请勿在配置真实 API Key 后截图分享给他人。

---

### 第三步：开始画图

在输入框中输入您的需求，例如：

```
画一个用户登录流程图
```

点击 **发送** 按钮，AI 将自动生成图表并显示在左侧画布上。

> 【截图位置：输入框和发送按钮】

## 高级功能

### 1. 模板管理

模板是保存常用 prompt 的功能，方便重复使用。

> 【截图位置：模板面板界面】

**创建模板：**

1. 点击输入框下方的 **书签图标** 📖+
2. 填写模板信息：
   - **提示词**：模板的核心内容
   - **标题**：模板名称（可选，默认取提示词前 20 字）
   - **描述**：模板说明（可选）
   - **置顶**：是否显示在列表顶部

3. 点击 **创建模板**

> 【截图位置：创建模板对话框】

**使用模板：**

在聊天面板的 "我的模板" 区域，点击模板即可发送。

> 【截图位置：模板列表和使用】

**模板管理功能：**
- 编辑：修改现有模板
- 复制：快速复制模板
- 删除：移除不需要的模板
- 置顶/取消置顶：调整模板排序
- 导入/导出：以 JSON 格式批量管理模板

---

### 2. 会话管理

每次对话都是一个独立的会话，包含完整的消息历史和图表状态。

> 【截图位置：会话历史面板】

**会话功能：**
- 自动保存：对话内容和图表会自动保存
- 会话切换：可以在不同会话之间切换
- 新建会话：点击 **新建对话** 按钮开始新会话
- 删除会话：删除不需要的历史会话
- 会话搜索：搜索历史会话内容

## 常见问题

### Q1: 页面打不开或加载慢

**可能原因和解决方案：**

| 问题 | 解决方案 |
|------|---------|
| 服务未启动 | 确认 `npm run dev` 已成功运行 |
| 端口错误 | 确认访问 `http://127.0.0.1:6002` |
| Draw.io 加载慢 | 等待加载完成，或检查网络连接 |
| 控制台有错误 | 按 F12 查看浏览器 Console 检查错误信息 |

---

### Q2: AI 模型不可用

**检查项：**

1. API Key 是否正确填写
2. Base URL 是否正确（如使用自定义端点）
3. Model ID 是否正确
4. 模型是否支持 tool calling 功能
5. 查看 Console 是否有错误信息

---

### Q3: 图表未生成或生成失败

**可能原因：**

| 原因 | 解决方案 |
|------|---------|
| 模型不支持 tool calling | 使用支持的模型（如 GPT-4o、Claude） |
| 网络问题 | 检查网络连接 |
| 输出被截断 | 尝试简化需求或增加 maxOutputTokens |
| XML 格式错误 | AI 会自动修复，如失败请重新生成 |

---

### Q4: 配置丢失

配置保存在浏览器的 localStorage 和 IndexedDB 中。

**可能原因：**
- 清理了浏览器数据
- 切换了浏览器或设备
- 使用了隐私模式

**解决方案：** 重新配置即可

---

### Q5: 如何导入已有的 draw.io 文件

Draw.io 嵌入式编辑器本身支持导入功能：

1. 在左侧画布区域直接操作
2. 使用 draw.io 的文件菜单导入 `.drawio`、`.xml` 或 `.svg` 文件
3. 导入后可以通过对话让 AI 继续编辑

---

### Q6: 支持哪些图表库的图形？

AI 支持以下图表库的图形：

| 类别 | 库名 |
|------|------|
| 云服务 | AWS、Azure、GCP、阿里云、Salesforce |
| 网络 | Cisco、Kubernetes、网络拓扑 |
| 业务 | BPMN、精益生产图 |
| 通用 | 流程图、基本形状、箭头、信息图 |
| UI | Android、Material Design |
| 工程 | 电路图、管道图、楼层平面图 |

## 技术说明

### 数据存储位置

所有数据存储在浏览器本地：

- **localStorage**：用户配置、会话 ID、设置选项
- **IndexedDB**：会话数据、模板数据

数据不会上传到服务器，完全本地化。

### 支持的浏览器

推荐使用现代浏览器：
- Chrome
- Edge
- Firefox
- Safari
